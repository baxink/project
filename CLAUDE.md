# CLAUDE.md

你正在管理一个船舶新造项目工作区。

## 目录约定

- 根目录下直接放每个船的项目目录。
- `README.md`：GitHub 首页总览看板。
- `dashboard.md`：内部详细版看板。
- `templates/project-template/`：新项目骨架。
- 标准阶段顺序：合同 → 设计 → 审图 → 建造 → 系泊 → 试航 → 发证 → 交船。
- `templates/naming-rules.md`：命名规则。

## 工作方式

- 优先读取目录名和 dashboard，再决定如何整理。
- 不要编造文件内容，只根据本地文件结构整理。
- 当前根目录下还没有任何船名项目目录；在用户没有明确要求前，不要预先创建示例船项目。
- 修改资料时，保持相对路径不变。
- 新增项目时，复制模板目录到根目录下的 `<项目名>/`。
- 更新进度时，同时更新 `README.md` 和 `dashboard.md`。README 放简洁总览，dashboard 放详细状态。两者根据同一项目数据生成，内容互补，不冲突。

## 文件命名

- 用短横线 `-`，少用空格。
- 禁止 `: ? * < > | " \ /`。
- 同一项目内命名风格保持一致。
- 阶段文件夹可按现场日期拆分成多个子文件夹。
- 进度字段统一使用 `N-M`。
- 默认步数：建造 10、系泊 3、试航 1、发证 1、交船 1。
- 默认从“建造”阶段开始展示进度，除非用户明确要求改成别的阶段。

## 看板规则

- 看板是手动维护的 Markdown。
- 不要使用 Markdown 表格。
- `README.md`：GitHub 首页，放简洁项目总览，由 LLM 根据项目数据生成。
- `dashboard.md`：内部详细看板，放完整项目区块、阶段状态和时间轴，由 LLM 维护。
- 每个项目使用一个独立文本区块，便于在记事本中直接阅读。
- 项目目录写相对路径。
- 进度必须同时保留两种表示：进度条 `█/░` 与步数 `N-M`。
- 若当前还没有任何船名项目目录，只保留空模板，不要伪造项目数据。
- 同步到 GitHub 时，只同步 `README.md` 和 `dashboard.md`，不要同步其他文件。
- 除 `README.md` 和 `dashboard.md` 外，其他项目目录、模板、配置和资料都视为授权文件，默认只在本地保存，不要推送到 GitHub。
