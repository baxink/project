# 详细项目看板

> 这是内部详细版看板；`README.md` 负责 GitHub 首页总览，这里保留更完整的项目状态。

## 使用方式

- 每个项目用一个独立区块表示。
- 项目区块里写纯文本目录路径，便于本地查找。
- 进度必须同时写两种：进度条 `█/░` + 步数 `N-M`。
- 默认从“建造”阶段开始展示进度；默认步数规则见 `templates/naming-rules.md`。

## 更新规则

- 新项目：在根目录复制模板生成船名文件夹，再在这里新增一个项目区块。
- 进度更新：同步改“当前进度”和对应阶段状态。
- 现场日期：记录在项目目录和对应阶段文件夹名中。
- 不用 Markdown 表格，避免不同软件里错位。
